## Summary

skills.sh is Vercel's open agent skills ecosystem, providing a CLI tool (`npx skills`) and directory/leaderboard platform for discovering, installing, and managing Agent Skills across 28+ AI coding assistants. The architecture follows the Agent Skills Specification (agentskills.io), using SKILL.md files with YAML frontmatter and a progressive disclosure pattern that loads skill content incrementally to optimize context window usage. Skills are filesystem-based packages containing instructions, scripts, and reference materials that extend agent capabilities without consuming context upfront.

## Key Findings

### Architecture Overview

The skills ecosystem consists of three interconnected components:

1. **CLI Tool (`npx skills`)** - Command-line interface for skill management
2. **skills.sh Platform** - Directory and leaderboard for skill discovery
3. **Agent Skills Specification** - Open standard defining the SKILL.md format

The system architecture is built around **progressive disclosure** - a design pattern where information loads in stages:

| Level | When Loaded | Token Cost | Content |
|-------|-------------|------------|---------|
| Level 1: Metadata | Always (startup) | ~100 tokens/skill | `name` and `description` from YAML frontmatter |
| Level 2: Instructions | When skill triggers | Under 5k tokens | SKILL.md body with instructions |
| Level 3+: Resources | As needed | Unlimited | Scripts, references, assets (executed via bash) |

### SKILL.md Format Specification

Every skill requires a `SKILL.md` file with YAML frontmatter:

```yaml
---
name: skill-name
description: Description of what this skill does and when to use it.
license: Apache-2.0  # optional
compatibility: Requires git, docker  # optional
metadata:  # optional
  author: example-org
  version: "1.0"
allowed-tools: Bash(git:*) Read  # optional, experimental
---

# Skill Instructions

[Markdown content with procedural knowledge, workflows, best practices]
```

**Field Requirements:**

| Field | Required | Constraints |
|-------|----------|-------------|
| `name` | Yes | 1-64 chars, lowercase alphanumeric + hyphens, must match directory name |
| `description` | Yes | 1-1024 chars, should include "Use when..." clause |
| `license` | No | License name or reference to bundled file |
| `compatibility` | No | Max 500 chars, environment requirements |
| `metadata` | No | Arbitrary key-value mapping |
| `allowed-tools` | No | Space-delimited pre-approved tools (experimental) |

### Directory Structure

```
skill-name/
├── SKILL.md          # Required - main instructions
├── scripts/          # Optional - executable code
│   └── extract.py
├── references/       # Optional - additional documentation
│   ├── REFERENCE.md
│   └── FORMS.md
└── assets/           # Optional - static resources
    └── templates/
```

### CLI Commands

The `npx skills` CLI provides comprehensive skill management:

| Command | Function |
|---------|----------|
| `npx skills add <source>` | Install skills from git repos |
| `npx skills find [query]` | Search skills interactively |
| `npx skills check` | Detect available updates |
| `npx skills update` | Upgrade all installed skills |
| `npx skills init` | Generate SKILL.md templates |
| `npx skills generate-lock` | Map installed skills to sources |

**Installation Examples:**

```bash
# List skills in repository
npx skills add vercel-labs/agent-skills --list

# Install specific skills to multiple agents
npx skills add vercel-labs/agent-skills -s frontend-design -a claude-code -a opencode

# Global installation (user-wide)
npx skills add vercel-labs/agent-skills -g

# CI/CD non-interactive mode
npx skills add vercel-labs/agent-skills --skill skill-name -g -a claude-code -y
```

**Installation Flags:**
- `-g, --global` - User-level installation
- `-a, --agent <name>` - Target specific platforms
- `-s, --skill <name>` - Install particular skills
- `-y, --yes` - Bypass prompts
- `--all` - Install everything without interaction
- `-l, --list` - Display available skills

### Installation Paths

Skills are installed to agent-specific directories:

| Agent | Project Path | Global Path |
|-------|--------------|-------------|
| Claude Code | `.claude/skills/<name>/` | `~/.claude/skills/` |
| OpenCode | `.opencode/skill/<name>/` | `~/.config/opencode/skill/` |
| Codex | `.codex/skills/<name>/` | `~/.codex/skills/` |
| Cursor | `.cursor/skills/<name>/` | `~/.cursor/skills/` |

### Discovery Mechanism

The CLI searches predefined locations in order:
1. Agent-specific directories (`.claude/skills/`, `.cursor/skills/`, etc.)
2. Project-local paths (walking up from CWD to git worktree root)
3. Global definitions (`~/.claude/skills/`, `~/.config/opencode/skill/`)

If standard paths yield no results, recursive searching activates.

### Update System

The update mechanism works through:

1. Reading `~/.agents/.skill-lock.json` for installed skills
2. POST request to `https://add-skill.vercel.sh/check-updates` with `skillFolderHash`
3. API computes fresh hashes and returns differences
4. Per-skill reinstallation for updates

The lock file uses v3 format; older versions require reinstallation.

### Advanced Features

**Installation Methods:**
- **Symlink** (recommended) - Unified updates across agents
- **Copy** - Independent agent instances

**Environment Variables:**
- `INSTALL_INTERNAL_SKILLS` - Enable hidden/WIP skills (`metadata.internal: true`)
- `DISABLE_TELEMETRY` or `DO_NOT_TRACK` - Opt-out of anonymous usage data
- Telemetry auto-disables in CI environments

**Platform-Specific Features:**
- `allowed-tools` - Restrict tool access (excluded from Kiro CLI, Zencoder)
- `context: fork` - Context forking (Claude Code only)
- Hooks support (Claude Code, Cline only)

### How Progressive Disclosure Works

1. **Startup**: Agent loads all skill metadata (name, description) into system prompt
2. **Request Matching**: When user request matches a skill description, agent reads full SKILL.md
3. **Resource Access**: Referenced files/scripts loaded only when needed
4. **Script Execution**: Scripts run via bash; only output enters context (code never loaded)

This enables:
- Installing many skills without context penalty
- Unbounded reference materials
- Efficient script execution
- On-demand file access

### Security Considerations

- Use skills only from trusted sources
- Malicious skills can invoke tools in harmful ways
- External URL fetching poses particular risk
- Thoroughly audit all bundled files before use
- Treat skill installation like software installation

### Ecosystem Competition

Multiple competing directories have emerged:
- skills.sh (Vercel)
- skillsdirectory.com
- skillsmp.com
- skillregistry.io
- openskills.space

### Community Feedback (from HN Discussion)

**Concerns Raised:**
- Installation opacity (unclear paths, upgrade/removal process)
- Ranking transparency (download counts favor Vercel's own skills)
- Inconsistent skill adoption by agents
- Lack of mature package management (versioning, dependencies)

**Workarounds Emerging:**
- Hooks to programmatically suggest relevant skills
- Shorter, focused skill designs
- Symlinks and Nix-based management for versioning

## Technical Implementation

The skills CLI is built with these core modules:

| Module | Function |
|--------|----------|
| `src/skills.ts` | Skill discovery and parsing |
| `src/installer.ts` | Symlink/copy installation |
| `src/source-parser.ts` | Parse git URLs, GitHub shorthand, local paths |
| `src/agents.ts` | Agent definitions and paths |
| `src/add.ts` | Add command logic |
| `providers/` | Remote providers (GitHub, HuggingFace, Mintlify) |

```bash
skills-ref validate ./my-skill
```
